Pathfinding Visualizer

About the algorithms

Breadth-First Search (BFS) is an uninformed search algorithm that explores all nodes at the current depth level before moving to the next level. It uses a queue (FIFO) for traversal and guarantees the shortest path in an unweighted graph. Its time complexity is O(V + E), and its space complexity is O(V) as it stores nodes in memory.
Depth-First Search (DFS) is another uninformed search algorithm that explores as far as possible along one branch before backtracking. It uses a stack (LIFO) or recursion for traversal. DFS does not guarantee the shortest path but can explore deep structures with less memory. Its time complexity is O(V + E), and its space complexity is O(V), mainly for the recursion stack.
A* Search is an informed search algorithm that uses both the cost to reach a node from the start (g(n)) and a heuristic estimate of the cost to the goal (h(n)) to decide the next node to explore. It uses a priority queue and is optimal when the heuristic is admissible (does not overestimate the true cost). Its time complexity depends on the heuristic but can approach O(V + E), and its space complexity is O(V), as it keeps track of nodes in memory.

How to use the website

The navigation bar provides several interactive options for using the pathfinding visualizer. The “Algorithms” menu allows you to choose between three algorithms: Depth-First Search (DFS), Breadth-First Search (BFS), and A* Search. DFS explores as far as possible along a single path before backtracking, BFS explores all neighboring cells level by level, and A* Search uses a heuristic to prioritize cells and ensures the shortest path when the heuristic is admissible. To use this menu, hover over “Algorithms” and select the algorithm you wish to visualize.
The “Add Walls” option lets you create obstacles on the grid. These walls act as barriers that the algorithm cannot pass through during its search. To add walls, hold down the mouse button and drag across the cells where you want to place the walls. Release the mouse button when you are done drawing.
The “Remove Walls” option allows you to remove any walls you have added to the grid. By holding down the mouse button and dragging over the cells you want to clear, you can make those cells accessible to the algorithm again.
The “Start Visualization” button begins the process of visualizing the selected algorithm. Before clicking this button, make sure you have selected an algorithm from the “Algorithms” menu, designated both a start and finish cell, and optionally added or removed walls to customize the grid. The visualization will demonstrate the algorithm’s search for the shortest path from the starting cell to the destination.
The “Select Start” option enables you to define the starting point of the algorithm. Click on a cell in the grid to mark it as the starting cell. Similarly, the “Select Finish” option allows you to select the destination cell. Click on a cell to designate it as the end point for the pathfinding process.
The “Clear Board” button resets the grid to its initial state by removing all walls, the start point, and the finish point. If you want to re-run an algorithm or make significant changes to the grid, it is essential to clear the board before proceeding.
